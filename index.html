What it does

Everything included.
Nothing extra.

Full directory tree

CLI-style ├── / └── hierarchy. Directories rendered before files at every level.

Language-aware blocks

60+ file types detected automatically. Every file wrapped in a properly fenced, syntax-tagged code block.

Two ways to run

Right-click any Explorer folder, or Ctrl+Shift+P → Tree Mapper: Generate Snapshot for the workspace root.

Smart ignore rules

.treeignore uses gitignore syntax. Auto-created on first run. Plus hardcoded exclusions for node_modules, .git, and binaries.

Auto-updates .gitignore

Detects a .git folder and automatically adds .tree/ and .treeignore — no duplicates ever.

Safe by default

Binary files detected by null-byte scan and skipped automatically. Files over the size limit are excluded with a clear notice.

How it works

Three steps. Done.

01

Install the extension

Click Install in VS Code below — it opens the extension panel directly in your editor, ready in one click.

02

Right-click a folder

In the Explorer panel, right-click any folder and select Tree Mapper: Generate Snapshot. A progress notification appears while it runs.

03

Get your snapshot

A Markdown file is saved to .tree/yyyy-mm-dd-hh-mm-ss.md. Click Open File in the notification to view it instantly.

Configuration

Settings

Open VS Code Settings (Ctrl+,) and search Tree Mapper.

Setting Default Description
treemapper.additionalIgnorePatterns [] Extra glob patterns to exclude, in addition to .treeignore. Useful for patterns like *.lock.
treemapper.maxFileSizeKB 2048 Files larger than this value (in KB) are skipped. Default is 2 MB. Lower it to speed up snapshots on large projects.
